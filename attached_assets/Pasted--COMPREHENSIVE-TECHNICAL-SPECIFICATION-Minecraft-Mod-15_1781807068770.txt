## 1. PROJECT OVERVIEW

Create a Minecraft NeoForge mod for **version 1.21.1** that adds **15 playable races**. Each race grants **8 active abilities**, passive traits, and unique penalties that fundamentally change gameplay. The mod must be:

- **Multiplayer-ready** (each player chooses independently via Curios accessory).
- **Persistent** (selections and progression survive death and world reload).
- **Configurable** via YACL in-game settings menu.
- **Performant** (minimal tick overhead, efficient GUI rendering).

## 4. CORE MECHANICS

### 4.1. Race Selection (One-time choice)

**Flow:**
1. Player joins the world for the first time (or if `race_data` is empty).
2. A chat message appears:
   ```
   §6⚔️ Добро пожаловать в мир Рас! §r
   §7Выбери свою расу – она определит твой путь.
   §a📖 Используй книгу «Том рас» (в инвентаре) или нажми §b[Клавиша] §aдля открытия меню выбора.
   §cВнимание: выбор расы окончателен!
   ```
3. Player receives a **written book** titled `"Том рас"` with 6 pages (intro + 15 races).
4. Clicking `[ВЫБРАТЬ ЭТУ РАСУ]` in the book OR selecting from GUI:
   - Sets `race_id` in Capability (1-15).
   - Gives the player their **Race Ring** (Curios accessory).
   - Removes the selection book (if in inventory).
   - Locks selection forever (cannot be changed).
5. **If player tries to select again** (e.g., using book after choice), display:
   ```
   §cТы уже выбрал расу! Твой путь предопределён.
   ```

**Technical implementation:**
- Use `Capability` attached to `Player` to store:
  - `int raceId` (0 = none, 1-15 = selected)
  - `boolean hasChosen` (once true, cannot be changed)
  - `int racePoints` (current unspent points)
  - `Map<Integer, Boolean> unlockedAbilities` (ability ID → unlocked)
  - `Map<Integer, Integer> spentPoints` (ability ID → points invested)
  - `float accessoryExp` (experience for ring level)
- Selection triggers **server-side** and syncs to client via custom packet.

### 4.2. Race Ring (Curios Accessory)

**Slot:** `curios:ring` (use existing ring slot).  
**Restriction:** Player can only wear **one** race ring at a time. If they try to equip a second, cancel the equip (or unequip the first).

**Item Properties:**
- `RaceRingItem` extends `Item` with `ICurioItem` interface.
- Each race has a separate item (or one item with NBT storing `raceId`).  
  **Recommendation:** Separate items per race for cleaner registry (e.g., `echo_golem_ring`, `mnemo_engineer_ring`, etc.).

**Tooltip (in Russian):**
```
§6Кольцо Эхо-Голема
§7Уровень: [level]
§7Опыт: [exp] / [next_level_exp]
§7Раса: Эхо-Голем
§7Пассив: звуковая чувствительность
§7Штраф: глухота, слабость на шерсти
```

**Leveling:**
- Each time the player uses an active ability, the ring gains `+1 exp`.
- Level = `floor(exp / 100)` (e.g., 100 exp = level 1, 200 = level 2, etc.)
- Max level: 10 (1000 exp).
- Level affects:
  - Ability damage: `base_damage * (1 + level * 0.05)` (max +50% at level 10).
  - Cooldown reduction: `base_cooldown * (1 - level * 0.02)` (max -20% at level 10).
  - Duration increase: `base_duration * (1 + level * 0.03)` (max +30% at level 10).

**GeckoLib Rendering:**
- No custom model – use standard Minecraft ring model (or simple 2D overlay).
- GeckoLib can render a static item model (no animations required). We'll use it for consistency, but it's optional.

### 4.4. Ability Tree (Progression Tree)

Each race has a **branching ability tree** with 3 tiers and 8 abilities total.

#### 4.4.1. Tree Structure

**Tier 1 (Base) – 0 points required:**
- 3 abilities (free, unlocked by default)

**Tier 2 (Intermediate) – 50 points required total:**
- 2 **branches** (A and B), each containing 2 abilities.
- Player must choose ONE branch and unlock both abilities in that branch.
- Cost per ability: 20 points.
- **Total tier cost:** 40 points (if they unlock both abilities in one branch).

**Tier 3 (Advanced) – 120 points required total:**
- 2 abilities (unlocked automatically after reaching 120 points).
- Cost per ability: 30 points.
- **Total tier cost:** 60 points.

**Total cost to unlock all 8 abilities:** 0 + 40 + 60 = 100 points (plus the 120 threshold for Tier 3).

**Visual representation in GUI:**
```
[Ability 1] [Ability 2] [Ability 3]   ← Tier 1 (free)
         ↓            ↓
    [Branch A]    [Branch B]         ← Tier 2 (50 pts)
    ↓        ↓    ↓        ↓
[Abil 4] [Abil 5] [Abil 6] [Abil 7]  ← Tier 2 abilities (20 pts each)
         ↓            ↓
    [Ability 8 (Advanced)]           ← Tier 3 (120 pts, 30 pts)
```

#### 4.4.2. JSON Configuration

Each race has a JSON file in `data/races/trees/<race_id>.json`:

```json
{
  "race_id": 1,
  "tier1": {
    "abilities": ["sound_burst", "echolocation", "silencer"],
    "cost": 0
  },
  "tier2": {
    "threshold": 50,
    "branches": {
      "branch_a": {
        "name": "Звуковая атака",
        "abilities": ["seismic_jump", "resonance"],
        "cost_per_ability": 20
      },
      "branch_b": {
        "name": "Защита",
        "abilities": ["vibration_shield", "sonic_breach"],
        "cost_per_ability": 20
      }
    }
  },
  "tier3": {
    "threshold": 120,
    "abilities": ["acoustic_accelerator", "sound_burst_enhanced"],
    "cost_per_ability": 30
  }
}
```

#### 4.4.3. Ability Unlock Logic
- Player must have **unspent points** to purchase an ability.
- Clicking an ability in GUI:
  - If not enough points: show `§cНедостаточно очков! Нужно: [cost]`
  - If unlocked: show `§aУже изучено!`
  - If branch requires other branch: show `§cВыбери ветку [A/B]!`
- When purchased:
  - Deduct points from `racePoints`.
  - Set `unlockedAbilities.put(abilityId, true)`.
  - Add ability to player's active ability list.

### 4.5. Abilities (8 per race)

Each ability is a **Java class** extending `AbstractAbility`.

**AbstractAbility.java:**
```java
public abstract class AbstractAbility {
    public final ResourceLocation id;
    public final String translationKey; // for GUI text
    public final int cooldownTicks;
    public final int baseCost; // hunger, health, etc.
    public final int levelScaling; // how much level increases effect

    public abstract void execute(Player player, Level level);

    public boolean canUse(Player player) {
        // Check race, cooldown, cost, etc.
    }

    public int getCooldown(Player player) {
        // Apply ring level reduction
    }
}
```

**Per-race ability classes** are instantiated and registered in `AbilityManager`.

#### 4.5.1. Ability Activation (Keybinds)
- **8 keybinds** (configurable in settings): `Ability 1` through `Ability 8`.
- Pressing a key activates the corresponding ability (if unlocked and off cooldown).
- Client sends packet to server to execute the ability.
- Server validates and runs `execute()`.

**Keybind default suggestions:**
- `G` – open progression tree GUI.
- `1-8` (or `R`, `F`, `C`, etc.) – activate abilities 1-8.

**Settings screen (YACL) allows rebinding all keys.**

### 4.10. Passive Effects & Penalties (Per Race)

Each race has passive traits and penalties that are applied **every tick** via the `onTick()` method in the race's data handler.

**Example: Эхо-Голем**
```java
// Every tick:
if (player.getRaceId() == 1) {
    // Passive: hunger drains 1.5x faster when sprinting
    if (player.isSprinting()) {
        player.getFoodData().addExhaustion(0.015f); // extra
    }
    // Penalty: standing on wool/slime reduces ability power
    Block block = player.getBlockStateOn().getBlock();
    if (block == Blocks.WOOL || block == Blocks.SLIME_BLOCK) {
        player.getPersistentData().putBoolean("echo_golem_weakened", true);
    } else {
        player.getPersistentData().putBoolean("echo_golem_weakened", false);
    }
}
```

**All penalties are implemented in the tick handler.**

## 6. TECHNICAL IMPLEMENTATION DETAILS

### 6.1. Capability System

**RaceData.java:**
```java
public class RaceData {
    private int raceId = 0; // 0 = none
    private boolean hasChosen = false;
    private int racePoints = 0;
    private boolean[] unlockedAbilities = new boolean[8]; // 8 abilities max
    private int[] spentPoints = new int[8]; // points invested per ability
    private int selectedBranch = -1; // -1 = none, 0 = branch A, 1 = branch B
    private float accessoryExp = 0f;
    private int accessoryLevel = 0;
    private long[] abilityCooldowns = new long[8]; // tick cooldowns

    // Getters/setters, sync via packets
}
```

**Provider:**
```java
@CapabilityInject(RaceData.class)
public static final Capability<RaceData> RACE_DATA_CAP = null;
```

**Attach to Player:**
```java
@SubscribeEvent
public static void attachCapability(AttachCapabilitiesEvent<Entity> event) {
    if (event.getObject() instanceof Player) {
        event.addCapability(new ResourceLocation(RacesMod.MOD_ID, "race_data"),
            new RaceDataProvider());
    }
}
```

### 6.2. Network Sync

**Packet:** `SyncRaceDataPacket` (server → client)
- Send on: join, level up, point change, ability unlock, cooldown update.
- Client receives and updates local `RaceData` (stored in client-side cache).

**Keybind Packet:** `AbilityActivationPacket` (client → server)
- Contains: ability ID (0-7).
- Server validates and executes ability.

### 6.3. Cooldown Management

Cooldowns stored in `RaceData.abilityCooldowns` (array of long ticks).
- On ability use: set to `System.currentTimeMillis() + (cooldownTicks * 50)`.
- In tick handler: if `cooldown > currentTime`, ability is unavailable.
- Cooldown reduction from ring level: `cooldownTicks * (1 - level * 0.02)`.

### 6.4. SmartBrainLib Integration

**SummonedMob.java:**
```java
public class SummonedMob extends Mob implements BrainEntity<SummonedMob> {
    private final Brain<SummonedMob> brain;

    @Override
    protected BrainProvider<SummonedMob> brainProvider() {
        return SmartBrainLib.createProvider(this, new SummonedMobBrain());
    }

    @Override
    public Brain<SummonedMob> getBrain() {
        return brain;
    }
}
```

**Brain Definition:**

```java
public class SummonedMobBrain extends Brain<SummonedMob> {

    }
}
```
